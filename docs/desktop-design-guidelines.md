# 桌面客户端设计规范

## 1. 文档目标

本文档用于指导当前项目后续桌面客户端界面设计与开发，确保整体体验在功能持续扩展时仍保持统一、简约、现代、美观，并同时支持浅色与深色主题。

适用范围：

- 桌面端主窗口
- 供应商配置相关界面
- 模型选择与参数配置界面
- 图片生成与结果展示界面
- 历史记录与本地存储管理界面
- 设置与主题切换界面

## 2. 设计定位

### 2.1 产品气质

整体视觉方向定义为：

- 冷静克制
- 专业现代
- 轻量高级
- 工具感明确，但不过于工程化

应避免：

- 过强的娱乐化视觉
- 过重的拟物设计
- 大面积荧光和高饱和撞色
- 杂乱的控制台风格
- 过度装饰导致的信息噪声

### 2.2 核心关键词

- Refined Utility
- Calm Precision
- Soft Depth
- Dense but Clear

### 2.3 设计记忆点

本产品的界面记忆点应来自以下组合：

- 大面积干净留白与半透明分层面板
- 结果图区域的沉浸式展示
- 精确克制的高亮色使用
- 浅深主题下都保持高级感的材质层次

## 3. 目标用户与使用场景

主要用户特征：

- 经常测试不同图像模型与供应商
- 关注配置效率、结果质量与历史回溯
- 长时间停留在工具内进行批量试验
- 对界面稳定性和信息可读性要求较高

主要使用场景：

- 维护多个供应商配置
- 拉取模型并测试不同参数组合
- 生成图片、放大图片、下载图片
- 回看历史结果与查找指定记录
- 管理本地历史图片存储目录

## 4. 整体设计原则

### 4.1 简约不是单薄

界面应尽量减少无效元素，但必须通过以下方式维持品质感：

- 清晰的层级关系
- 稳定的网格与间距系统
- 精准的色彩对比
- 细腻的状态反馈

### 4.2 先信息，再装饰

所有视觉设计首先服务于以下目标：

- 快速理解当前上下文
- 快速定位关键操作
- 快速判断系统状态

### 4.3 强化主任务路径

主任务路径必须始终最突出：

1. 选择供应商
2. 拉取模型
3. 配置参数
4. 生成结果
5. 浏览与保存结果

### 4.4 高级感来源于克制

动画、渐变、阴影、玻璃效果都可以使用，但必须遵守：

- 只在关键区域使用
- 不影响信息清晰度
- 不制造持续性干扰

## 5. 信息架构建议

桌面端建议采用三层结构，而不是单页堆叠所有能力。

### 5.1 一级结构

- 工作台
- 历史记录
- 设置

### 5.2 二级结构建议

工作台包含：

- 供应商
- 模型
- 参数
- 生成结果

历史记录包含：

- 历史列表
- 筛选与搜索
- 记录详情

设置包含：

- 通用设置
- 外观主题
- 本地存储
- 关于

### 5.3 推荐布局

推荐桌面端主界面采用：

- 左侧导航栏
- 中部主工作区
- 右侧上下文面板

建议结构如下：

```text
+------------------------------------------------------+
| 侧边导航 | 主工作区                  | 结果/详情侧栏 |
|         |                           |              |
| 工作台   | 供应商 / 模型 / 参数       | 当前结果      |
| 历史记录 |                           | 或历史详情     |
| 设置     |                           |              |
+------------------------------------------------------+
```

这样比当前双栏更适合桌面端长期扩展。

## 6. 视觉系统

### 6.1 色彩策略

配色应保持“低饱和基底 + 小面积高识别强调色”的原则。

#### 深色主题

建议方向：

- 背景基底：深墨蓝灰，不用纯黑
- 面板层：低透明度冷灰蓝
- 文字主色：偏冷白
- 强调色：青绿偏蓝
- 次强调色：琥珀金

参考角色：

- 主背景：`#0B1016` 到 `#121922`
- 面板背景：`rgba(17, 24, 34, 0.78)`
- 主文字：`#ECF2FA`
- 次文字：`#9DAEC4`
- 强调色：`#49C2B8`
- 警示色：`#F2A94A`
- 错误色：`#E36D6D`

#### 浅色主题

建议方向：

- 背景基底：暖白或矿物白，不用纯白
- 面板层：奶油灰白
- 文字主色：深棕灰或深石墨
- 强调色：深青色
- 次强调色：铜棕色

参考角色：

- 主背景：`#F5F1E8` 到 `#FCFAF6`
- 面板背景：`rgba(255, 250, 242, 0.84)`
- 主文字：`#2C241B`
- 次文字：`#6C5C4B`
- 强调色：`#0D7268`
- 警示色：`#AF6A1C`
- 错误色：`#C84B4B`

### 6.2 色彩使用规则

- 强调色只用于当前选中、主按钮、焦点态、关键标签
- 警示色仅用于容量接近上限、注意提示、次级高风险提醒
- 错误色仅用于失败状态、删除操作、不可逆提示
- 成功色要比强调色更少出现，避免界面泛绿

### 6.3 背景层次

建议使用三层背景：

- 最底层：柔和渐变背景
- 中间层：微弱光斑或径向气氛
- 上层：半透明内容面板

要求：

- 不使用强纹理
- 不使用高对比花纹
- 不使用明显噪点覆盖主要内容

## 7. 字体规范

### 7.1 字体策略

桌面端应采用“双字体策略”：

- 标题字体：有一定识别度的现代几何或人文无衬线
- 正文字体：高可读性的中文优先字体

建议组合方向：

- 英文与数字标题：`Avenir Next` 或 `Bahnschrift`
- 中文正文：`PingFang SC`、`Microsoft YaHei UI`

如果后续引入可分发字体，再单独评估许可证与打包体积。

### 7.2 字号层级

建议层级：

- 窗口主标题：24
- 模块标题：18
- 面板标题：14
- 正文：13 到 14
- 辅助说明：12
- 标签与状态：11 到 12

### 7.3 排版要求

- 标题避免过粗，推荐 `600`
- 正文字重优先 `400` 到 `500`
- 行高保持宽松，正文建议 `1.5`
- 英文和数字在设置、参数、尺寸中要保证对齐感

## 8. 空间与圆角系统

### 8.1 间距系统

建议基础间距采用 4 的倍数体系：

- 4
- 8
- 12
- 16
- 20
- 24
- 32

### 8.2 圆角系统

建议统一圆角等级：

- 输入框与小按钮：12
- 卡片与面板：18 到 22
- 弹窗与大容器：24 到 28
- 胶囊标签：999

### 8.3 阴影系统

阴影应柔和、扩散、偏大半径，不使用厚重硬阴影。

建议分三档：

- 面板阴影：轻
- 弹出层阴影：中
- 图片预览层阴影：重

## 9. 组件设计规范

### 9.1 导航栏

左侧导航栏需要满足：

- 固定宽度
- 明确当前激活项
- 支持图标 + 文字
- 底部可放主题切换与设置入口

导航气质应更像专业创作工具，而不是网页菜单。

### 9.2 面板

所有核心功能采用面板化组织。

面板规范：

- 标题清晰
- 信息密度适中
- 内部使用统一间距
- 避免在一个面板中承载过多不同类型操作

### 9.3 表单

表单应强调录入效率与状态明确性。

规则：

- 标签固定在输入框上方
- 占位文案只做提示，不替代标签
- 焦点态需要清晰，但不刺眼
- 错误提示紧贴字段
- 敏感字段支持显隐切换

### 9.4 按钮

按钮分为四类：

- 主按钮：最重要操作，如“生成图片”
- 次按钮：辅助操作，如“拉取模型”
- 轻按钮：卡片内操作，如“回显”“下载”
- 危险按钮：删除、清空

规范：

- 主按钮使用强调色渐变或纯色高亮
- 次按钮使用低对比实体面
- 危险按钮默认低调，仅 hover 后强化风险提示

### 9.5 标签与状态

标签用于表达：

- 模型类型
- 供应商
- 当前模式
- 容量状态
- 默认项

要求：

- 体积小
- 视觉简洁
- 不抢主内容

### 9.6 图片卡片

结果图卡片必须是桌面端的重点组件。

要求：

- 图片展示优先级高于说明文字
- 顶部显示简洁元信息
- 底部操作区固定且易点击
- 支持 hover 高亮、点击预览
- 支持显示尺寸、模型、耗时、放大状态

### 9.7 历史记录项

历史项应兼顾浏览效率与信息完整性。

建议结构：

- 左侧缩略图
- 中间主信息
- 右侧操作区

必须可快速识别：

- 时间
- 模型
- 生成模式
- Prompt 摘要
- 文件体积

### 9.8 设置项

设置页不应只是普通表单堆叠，建议按“设置卡片”组织：

- 标题
- 一句话说明
- 当前值
- 操作按钮

历史目录这类系统级设置应额外显示：

- 当前路径
- 打开目录
- 更改目录
- 存储占用

## 10. 主题规范

### 10.1 主题目标

浅色和深色主题必须满足：

- 同样高级
- 同样清晰
- 不只是简单反色

### 10.2 深色主题原则

- 更适合长时间生成与看图
- 图片区域对比更强
- 结果展示沉浸感更好

深色主题要避免：

- 纯黑背景
- 过亮青色
- 文字发灰导致疲劳

### 10.3 浅色主题原则

- 更适合配置录入与历史查阅
- 视觉要轻，但不能发白发飘
- 用暖白增强质感

浅色主题要避免：

- 冷白办公软件感
- 线条过浅导致边界丢失
- 颜色太淡导致无重点

### 10.4 主题切换体验

- 切换时支持 150ms 到 220ms 柔和过渡
- 不出现整页闪烁
- 图片区域不做突兀反相动画

## 11. 动效规范

### 11.1 动效原则

动效服务于理解，不服务于炫技。

### 11.2 推荐动效场景

- 应用首次进入时面板轻微上浮淡入
- 左侧导航切换时内容区平滑切换
- 卡片 hover 轻微抬升
- 生成中状态显示精致加载反馈
- 弹窗与图片预览使用柔和缩放淡入

### 11.3 时长建议

- Hover：120ms 到 180ms
- 面板过渡：180ms 到 240ms
- 弹窗出现：220ms 到 280ms
- 主题切换：180ms 左右

### 11.4 禁止项

- 大幅弹跳
- 高频闪烁
- 多处同步强动效
- 无意义的持续呼吸动画

## 12. 关键页面设计要求

### 12.1 工作台

工作台是最高频页面，应突出效率与聚焦。

建议布局：

- 左列：供应商与模型
- 中列：Prompt、模式、参数
- 右列：当前结果

主按钮“生成图片”必须始终明显可见。

### 12.2 历史记录

历史记录页应偏“资产浏览”而不是“普通列表”。

应具备：

- 搜索
- 模型筛选
- 模式筛选
- 时间分组或按时间倒序
- 详情回显

后续如果记录很多，建议加入：

- 网格 / 列表切换
- 按日期折叠

### 12.3 设置页

设置页应更安静、更系统化。

推荐模块：

- 外观
- 供应商与默认行为
- 放大服务
- 历史目录
- 存储占用

### 12.4 图片预览层

大图预览必须具备：

- 中央聚焦
- 背景遮罩
- 简洁关闭动作
- 图片信息最小化展示

可选增强：

- 原图尺寸
- 快速下载
- 打开所在目录

## 13. 可用性与无障碍要求

- 所有点击区域最小高度不低于 36
- 主操作按钮不低于 40
- 文字与背景对比满足可读性要求
- 焦点态清晰可见
- 不仅依赖颜色表达状态
- 图标按钮必须有 tooltip 或文字说明

## 14. 开发落地建议

### 14.1 设计令牌

后续实现应优先建立统一设计令牌：

- 颜色变量
- 字体变量
- 圆角变量
- 阴影变量
- 动效变量
- 间距变量

### 14.2 组件优先级

建议先统一这些基础组件：

- AppShell
- Sidebar
- Panel
- Button
- Input
- Select
- StatusBadge
- ResultCard
- HistoryCard
- SettingCard
- Modal

### 14.3 代码层原则

- 所有主题通过变量驱动
- 不在组件内散落硬编码颜色
- 状态类命名统一
- 先搭骨架，再做视觉精修

## 15. 当前项目的具体视觉建议

结合当前功能，后续桌面端建议采用：

- 深色主题作为默认主题
- 浅色主题作为高可读备选
- 结果图区域占据更高视觉权重
- 设置页单独拆出，而不是继续堆在工作台顶部
- 历史记录独立成导航页
- 供应商配置从“表单块”升级为“可管理资源卡片”

一句话方向：

把当前页面从“功能堆叠型网页工具”升级为“专业图像生成桌面工作台”。
